Here are the basic CSS Grid properties used in this app:

Grid options:

width and height - pretty self explanatory, it's just height and width of the grid container.

columns - full property name is "grid-template-columns". It tells the grid container on how many columns should it place the grid items. You can put there any value in px, %, rem, em, vh or vw.

rows - full property name is "grid-template-rows". It tells the grid container on how many rows should it place the grid items. You can put there any value in px, %, rem, em, vh or vw.

column gap - full property name is "grid-column-gap". It tell the grid how big should be the gutter between columns. You can put there any value in px, %, rem, em, vh or vw.

column gap - full property name is "grid-row-gap". It tell the grid how big should be the gutter between rows. You can put there any value in px, %, rem, em, vh or vw.

grid-auto-flow - Grid has preety great auto-placement algorithm, and "grid-auto-flow" property defines how should it act. If you set it to "column" it will place new grid items in availble columns or create a new one and place it there. "Row" will place new grid items in availble rows or create a new one and place it there. "Dense" will try to fit new items in availble spots, also changing the order of grid items, so be careful with that.

Grid item options:

grid-column-start - specifies in which column the item will start. Only integers are allowed.

grid-column-end - specifies in which column the item will end. Only integers are allowed.

grid-row-start - specifies in which row the item will start. Only integers are allowed.

grid-row-end - specifies in which row the item will end. Only integers are allowed.

Other options:

CSS Grid offers a lot of more options than shown in this playground. Options like "align-items", "justify-items", "align-content", "justify-content" works preety much the same as in Flexbox, so if you know that you should also be able to use it in your grid, but there is also a new value of these properties - "space-evenly", which is very cool - it evenly adds space between items and container edges. There are also short-hand properties like "grid-template", in which you can even name your rows and columns, "grid-gap" in which you specify both column and row gaps. There is also just "grid" in which you specify all of the above, but for me it makes the code less readable, but if you like it, use it.

Gris items also have some more properties, like "grid-area" in which you tell the item which named areas of your grid should it take, and of course "align-self" and "justify-self". You can also use short-hand properties "grid-column" and "grid-row" to specify both the start and the end of yout item.
